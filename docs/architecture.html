Multi-Agent Review

3개의 전문 에이전트가 동시에 PR diff를 분석합니다. 각 에이전트는 동적 합성된 프롬프트로 구동됩니다.

Prompt Composition

각 에이전트의 system prompt는 3개 레이어를 --- 구분자로 연결하여 동적 합성됩니다.

1
Agent 기본 프롬프트 필수
에이전트의 역할과 범용 규칙. FE/BE 무관하게 적용됩니다.
agents/bug-detector.md agents/security-checker.md agents/code-quality.md agents/synthesizer.md
+
2
Team 프롬프트 선택
팀 공통 코딩 규칙. 없으면 경고 로그 후 스킵합니다.
teams/frontend.md teams/backend.md (향후)
+
3
Stack 프롬프트 (복수 가능) 선택
기술 스택별 전문 규칙. 쉼표로 여러 개 지정할 수 있습니다.
stacks/nextjs-app-router.md stacks/react-performance.md stacks/tanstack-query.md stacks/tailwindcss.md stacks/turborepo.md
Customization

대상 레포의 caller workflow에서 아래 값들을 설정합니다.

team
팀 식별자 — prompts/teams/{team}.md 로드
"frontend" | "backend"
stacks
기술 스택 (쉼표 구분) — prompts/stacks/{stack}.md 로드
"nextjs-app-router,tanstack-query,tailwindcss"
exclude_patterns
리뷰 제외 파일 패턴 (쉼표 구분)
"*.test.ts,*.stories.tsx,pnpm-lock.yaml"
review_model
AI 모델 오버라이드
"auto" | "claude-sonnet-4-6" | "claude-opus-4-6"
Usage Example

대상 레포에 이 파일 하나만 추가하면 됩니다.

.github/workflows/senior-review.yml # FE 레포 예시 name: Senior Code Review on: pull_request: types: [opened, synchronize, ready_for_review] jobs: review: if: github.event.pull_request.draft == false uses: aptimizer-co/fe-senior-reviewer/.github/workflows/review-executor.yml@main with: team: "frontend" stacks: "nextjs-app-router,react-performance,tailwindcss,tanstack-query" exclude_patterns: "*.test.ts,*.test.tsx,*.stories.tsx,pnpm-lock.yaml,*.md" secrets: ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }} REVIEWER_TOKEN: ${{ secrets.REVIEWER_TOKEN }} # BE 레포 예시 (향후) # team: "backend" # stacks: "nestjs,prisma,postgresql"
Error Handling
시나리오 동작
🔴 agents/*.md 파일 없음 에러 발생 — 리뷰 중단
🟠 teams/*.md 파일 없음 경고 로그 → 해당 프롬프트 없이 진행
🟠 stacks/*.md 파일 없음 경고 로그 → 해당 프롬프트 없이 진행
🟠 에이전트 1개 실패 Promise.allSettled — 나머지 결과로 계속 진행
🟢 인라인 코멘트 실패 전체 findings를 텍스트 요약 코멘트에 포함
🟢 team/stacks 미지정 에이전트 기본 프롬프트(범용)만으로 리뷰
